# 附录 A：核心关键字与运算符速查（Rust ↔ C++）

## 说明

这份速查表不追求完全一一对应，而是提供阅读时最常见的“语义近邻”。看到 C++ 关键字时，你可以先借助熟悉的 Rust 概念建立第一层理解，再进一步检查差异。

## 基础类型与绑定

| Rust | C++ | 阅读提示 |
| --- | --- | --- |
| `let` | 普通局部变量声明 | C++ 默认可变 |
| `let mut` | 非 `const` 变量 | 可变是默认状态 |
| 不可变绑定 | `const` | 需区分顶层与底层 `const` |
| `&T` | `const T&` | 近似只读引用，但无借用检查 |
| `&mut T` | `T&` | 不保证独占访问 |
| 原始指针 | `T*` / `const T*` | 允许为空，生命周期需手动判断 |

## 所有权与资源管理

| Rust | C++ | 阅读提示 |
| --- | --- | --- |
| `Box<T>` | `std::unique_ptr<T>` | 唯一拥有者 |
| `Rc<T>` / `Arc<T>` | `std::shared_ptr<T>` | 共享所有权 |
| `Weak<T>` | `std::weak_ptr<T>` | 非拥有观察者 |
| `Drop` | 析构函数 + RAII | 释放时机依作用域与对象寿命 |
| move | `std::move` + 移动构造/赋值 | `std::move` 本身不执行移动 |

## 控制流

| Rust | C++ | 阅读提示 |
| --- | --- | --- |
| `if` 表达式 | `if` 语句 | C++ `if` 不是值表达式 |
| `match` | `switch` / `std::visit` | 通常缺少穷尽性保障 |
| `loop` | `for(;;)` / `while(true)` | 语言风格不同 |
| `for x in xs` | 范围 `for` | 底层依赖迭代器 |

## 泛型与抽象

| Rust | C++ | 阅读提示 |
| --- | --- | --- |
| 泛型函数 | 函数模板 | 实例化通常发生在使用点 |
| 泛型类型 | 类模板 | 常见于容器与工具类型 |
| trait bound | Concepts / `enable_if` | 旧代码中可能非常隐晦 |
| trait object | `virtual` + 基类指针/引用 | 动态分发依赖虚表 |

## 错误处理

| Rust | C++ | 阅读提示 |
| --- | --- | --- |
| `Option<T>` | `std::optional<T>` | 语义较接近 |
| `Result<T, E>` | `std::expected<T, E>` | C++23 才标准化 |
| `panic!` | `throw` 并不等价 | 异常可传播、捕获、跨层清理 |

## 其他高频关键字

| C++ 关键字/写法 | 近似理解 |
| --- | --- |
| `virtual` | 启用动态分发 |
| `override` | 明确重写虚函数 |
| `final` | 禁止继续重写或继承 |
| `explicit` | 防止隐式转换 |
| `constexpr` | 编译期可求值能力 |
| `noexcept` | 声明不抛异常 |
| `friend` | 授权外部访问私有成员 |
| `typename` | 模板上下文中的类型提示 |
| `using` | 别名或名字引入，不等于模块导入 |

## 不要误会成什么

| 看到的写法 | 不要直接误会成 | 更接近的理解 |
| --- | --- | --- |
| `T&` | Rust 的 `&mut T` | 只是引用，不保证独占 |
| `const T&` | Rust 的 `&T` 完整等价物 | 近似只读观察，但无借用检查 |
| `std::move(x)` | Rust 里的 move 关键字 | 只是把 `x` 转成可移动来源 |
| `std::optional<T>` | 一定是错误返回 | 也可能只是“值可能缺失” |
| `throw` | `panic!` | 更接近可传播、可捕获的错误通道 |
| `using` | 模块导入 | 名字引入或别名定义 |
| `virtual` | trait 的完整等价物 | 只是动态分发的一部分机制 |
| `std::variant` | Rust `enum` 完整等价物 | 接近代数数据类型，但约束更弱 |
