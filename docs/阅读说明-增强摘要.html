阅读说明：增强摘要

这页不是任务清单，而是给读者的简短说明。

如果你刚进入这套文档，想快速知道它现在怎么用、哪些地方已经补强，可以先看这里。

这套文档现在适合怎么用

  • 如果你想系统读：从 书籍首页 或 全书总索引 开始
  • 如果你已经在项目里卡住：优先用 全书总索引 里的“按问题快速定位”
  • 如果你想把它当项目阅读手册：优先看 第四部分
  • 如果你只是临时查一个点：直接去 附录

这轮主要增强了什么

1. 实战章节更可执行了

  • 第12章 微型 JSON 解析库 现在不只是提纲，还补了目录树、关键类型、调用链和生命周期标注。
  • 第14章 驾驭大型项目 现在包含 30 分钟首轮摸底流程、阅读产物模板和风险标注方式。

2. 工程高风险主题更落地了

  • 第6章 错误处理 补了 ABI / C 边界、禁用异常项目和错误翻译层。
  • 第8章 模板基础 补了从签名反推约束、拆 enable_if / traits 链和模板报错回溯法。
  • 第11章 未定义行为 补了高频红旗模式表和低层内存代码审查问题。

3. 全书模板更统一了

大部分主章节现在都有更统一的阅读结构：

  • 适合谁读
  • 最小示例或核心对照
  • 常见误判
  • 阅读时先问什么
  • 练习
  • 检查清单

这意味着你在不同章节之间切换时，阅读动作会更稳定。

4. 导航页更像真正的导航了

  • 书籍首页 现在有“快速进入”和“按场景选路线”
  • 全书总索引 现在有“按问题快速定位”和“按投入时间阅读”
  • 每个部分首页现在都补了“适合谁先读”“怎么读这一部分”“读到卡住时跳哪里”
  • 每个章节 index.md 都补了返回部分首页和全书总索引的入口
