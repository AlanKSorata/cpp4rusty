第三部分：激战区——深度、危险与生存技能

这一部分对应阅读 C++ 时最容易产生认知负担的内容：模板、移动语义、宏、未定义行为。它们不是可选知识，而是在真实工程中反复出现的核心机制。

适合谁先读

  • 已经进入高密度 C++ 代码区，正在和模板、宏、生命周期风险正面碰撞
  • 需要的不再是“这是什么语法”，而是“我怎么在危险区域里保持判断力”
  • 开始做 Review、排障或阅读底层实现

阅读目标

读完这一部分，你应该能做到：

  • 从复杂模板签名中抓出约束、参数与实例化线索
  • 理解移动语义、拷贝控制与 RAII 的实际阅读价值
  • 在宏密集代码中保持结构感，不被预处理层干扰
  • 建立识别 UB 风险的基础审查习惯

怎么读这一部分

如果你是按顺序阅读，建议 8 → 9 → 10 → 11 连着看，因为它们会逐步把“复杂语法”“资源语义”“宏层噪声”“高风险审查”串起来。

如果你是带着问题来：

  • 模板和长报错：先读 第8章
  • RAII、拷贝、移动：先读 第9章
  • 宏、条件编译：先读 第10章
  • UB、悬垂、数据竞争：先读 第11章
