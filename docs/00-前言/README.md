# 前言

## 本章导读

如果你已经会写 Rust，再去读 C++，最强烈的感受通常不是“完全陌生”，而是“很多东西看起来认识，但总觉得不安全”。这正是本书存在的原因。

Rust 与 C++ 共享大量系统编程领域的目标：高性能、零成本抽象、资源管理、泛型、显式控制。但二者在**安全边界、默认约束、语言承诺**上差异极大。Rust 倾向于让编译器尽可能早地阻止错误；C++ 则更常把决定权交给程序员、库设计者和工程约定。

因此，Rust 开发者阅读 C++ 的难点，不是记不住语法，而是容易把 Rust 的直觉错误投射到 C++ 代码上。

## 为什么你需要这本书

很多资料教你“如何写 C++”，却很少直接回答另一个更现实的问题：

> 当你必须进入一个已有的 C++ 工程时，应该先看什么、警惕什么、如何避免误读？

真实工作里，更常见的场景是：

- 你要排查线上问题，调用栈落到 C++ 库里。
- 你在混合语言项目中，需要理解 C++ 模块对 Rust 的影响。
- 你要 review 一段 C++ 改动，但并不打算长期主写 C++。
- 你要接手老代码、第三方 SDK、跨平台基础设施或性能关键路径。

这时，**阅读能力比写作能力更优先**。

## 阅读优先：学会看懂，而非从头编写

本书默认你不是在系统学习一门新语言，而是在建立一种高效率的阅读模式。重点不在于覆盖全部语法细节，而在于回答以下问题：

- 这段代码里的对象是谁拥有的？
- 生命周期在哪里结束？
- 哪些地方可能有未定义行为？
- 这里为什么用了 `virtual`、模板、宏或智能指针？
- 某个 API 看起来像 Rust 的对应物，但语义差别到底在哪里？

当你能稳定回答这些问题时，你已经具备了阅读真实 C++ 工程的核心能力。

## 如何使用本书

建议采用“对比学习 + 代码导航”的方式阅读：

- **对比学习**：把 Rust 中熟悉的概念当作锚点，例如 `Option<T>`、`Box<T>`、trait object、`Drop`、借用与所有权。
- **代码导航**：读完一个概念后，立刻去实际项目里寻找对应模式，例如 `std::unique_ptr`、`std::variant`、`override`、`noexcept`、`#ifdef`。
- **优先抓语义**：不要先陷入细枝末节的语法争论，先判断内存、所有权、分发方式、异常与资源释放。
- **保留怀疑**：在 Rust 中“理所当然安全”的东西，在 C++ 中往往需要你额外确认。

## 版本约定

- 本书以 **C++17** 为基线。
- 对 **C++20 / C++23** 的内容会明确标注。
- 如果某个特性只在更新标准中可用，会同时说明它在阅读旧项目时可能缺席。

## 示例代码与配套资源

建议你在阅读过程中准备一套最小工具环境：

- 支持 C++17 的编译器
- CMake
- clangd 或等效 LSP
- 一个可快速跳转定义、查看引用、浏览头文件的编辑器

这些工具不会替你理解代码，但会大幅降低定位结构与验证猜测的成本。
