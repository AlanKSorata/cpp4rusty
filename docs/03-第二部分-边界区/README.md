# 第二部分：边界区——似而不同的惯用法

这一部分开始进入 Rust 开发者最容易“望文生义”的区域。很多机制表面上有对应物，但实际语义和工程后果差异很大。

## 适合谁先读

- 已经能读一些 C++，但在继承、异常、`variant` 上总觉得“看懂了又没完全看懂”
- 正在项目里处理语义边界问题，而不是基础语法问题
- 需要校正“看起来像 Rust，所以我以为自己懂了”的误判

## 阅读目标

读完这一部分，你应该能做到：

- 看懂典型的类层次、虚函数、多态接口与继承风险
- 区分异常、返回值、错误码在工程中的不同角色
- 理解 `variant` 与 Rust `enum` 在表达能力和穷尽性上的重要差异

## 怎么读这一部分

这一部分更适合带着问题读，而不是完全顺序通读：

- 如果你在类层次和虚函数里迷路，先读 [第5章](./第5章-类与面向对象/README.md)
- 如果你在异常和返回值策略里迷路，先读 [第6章](./第6章-错误处理/README.md)
- 如果你在 `variant` 和 visitor 里迷路，先读 [第7章](./第7章-代数数据类型/README.md)

如果没有明确问题，仍建议按 5 → 6 → 7 顺序读，因为它正好覆盖三类最常见的“似而不同”机制。
