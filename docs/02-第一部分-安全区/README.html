第一部分：安全区——可预测的相似地带

这一部分讨论的是 Rust 开发者最容易建立映射关系的内容。它们未必绝对安全，但通常更适合作为阅读 C++ 的起点。

适合谁先读

  • 第一次进入 C++ 仓库，不知道该从哪里建立基本阅读能力
  • 已经能看懂一点语法，但还会在声明、指针、容器上反复误判
  • 想先抓住“比较像 Rust”的区域，再逐步进入危险地带

章节目录

  • 第1章 编译与项目结构：从 Cargo 到 CMake

  • 第3章 智能指针：所有权在 C++ 中的投影
  • 第4章 字符串、容器与迭代器：集合生态的对比

阅读目标

读完这一部分，你应该能做到：

  • 识别一个基本 C++ 项目的构建与文件组织方式
  • 看懂常见变量声明、const、引用与指针的语义轮廓
  • 区分 unique_ptr、shared_ptr、weak_ptr 的角色
  • 理解 string、vector、map、迭代器在阅读中的关键风险点

怎么读这一部分

建议按章顺序阅读，不要跳读太多。原因是这四章正好构成一条基础路径：

  1. 先用 第1章 建立“仓库从哪里开始看”的方向感。
  2. 再用 第2章 建立“看到一行声明时如何拆”的能力。
  3. 然后用 第3章 把“谁拥有谁”这条主线找回来。
  4. 最后用 第4章 把容器、视图和失效风险接上。

读到卡住时跳哪里

  • 卡在 const、引用、auto：跳到 第2章目录
  • 卡在所有权与生命周期：跳到 第3章目录
  • 卡在 string_view、vector、迭代器：跳到 第4章目录
