《从 Rust 到 C++：代码阅读完全指南》

面向 Rust 开发者的 C++ 代码阅读指南。

本书以 C++17 为基础，对必要的 C++20 / C++23 特性做额外标注，目标不是把你训练成传统意义上的 C++ 写手，而是帮助你在真实工程里更快、更安全地读懂 C++。

阅读入口

  • 前言
  • 序章：安全模式切换——从 Rust 到 C++ 的心智准备
  • 第一部分：安全区——可预测的相似地带
  • 第二部分：边界区——似而不同的惯用法
  • 第三部分：激战区——深度、危险与生存技能
  • 第四部分：实战阅读——从案例到大型工程
  • 附录
  • 阅读说明：本轮增强摘要
  • 文档增强清单
  • 全书总索引

如果你只想快速进入

  • 想先建立心智模型：从 序章 开始
  • 想先拿回“能读懂常见 C++”的基本能力：从 第一部分 开始
  • 已经在项目里卡在异常、继承、variant：直接去 第二部分
  • 已经在读模板、宏、生命周期高风险代码：直接去 第三部分
  • 想把前面知识带回真实工程：直接去 第四部分
  • 想现场速查：直接去 附录

适合谁

  • 已经熟悉 Rust，但需要阅读 C++ 项目代码的人
  • 需要在混合语言项目中排查问题、Review 代码或理解第三方库的人
  • 想快速建立 C++ 阅读心智模型，而不是先系统学习全部语言细节的人

阅读策略

建议按下面的顺序阅读：

  1. 先看序章，建立“C++ 代码默认处于全局 unsafe 环境”这一前提。
  2. 再进入“安全区”，优先理解项目结构、基础类型、智能指针、容器与迭代器。
  3. 然后进入“边界区”，识别那些看上去像 Rust、实际上语义差异很大的机制。
  4. 最后阅读“激战区”和“实战阅读”，形成工程级阅读能力。

按场景选路线

场景 1：我刚开始读 C++，只想先别误判

建议顺序：

  1. 序章
  2. 第1章 编译与项目结构
  3. 第2章 类型系统与基础语法
  4. 第3章 智能指针
  5. 第4章 字符串、容器与迭代器

场景 2：我已经在项目里，被具体语义问题卡住

优先跳转：

  • 继承、多态、虚函数： 第5章
  • 异常、optional、expected： 第6章
  • variant、visitor： 第7章
  • 模板报错和泛型库： 第8章
  • 资源、移动、RAII： 第9章
  • 宏、条件编译： 第10章
  • UB、悬垂、竞争： 第11章

场景 3：我想把这本书当项目阅读手册

建议顺序：

  1. 第12章 微型 JSON 解析库
  2. 第13章 常见 C++ 工程模式
  3. 第14章 驾驭大型项目
  4. 附录 E 阅读工具包配置指南
