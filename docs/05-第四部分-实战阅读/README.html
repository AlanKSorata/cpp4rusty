第四部分：实战阅读——从案例到大型工程

这一部分把前面的概念带回真实项目阅读。重点不再只是“某个语言点是什么意思”，而是“进入一段陌生 C++ 工程后应该如何建立全局理解”。

适合谁先读

  • 你已经理解前面的大部分概念，但还不会把它们组合成项目阅读路径
  • 你想把这本书当成“如何读仓库”的实战手册
  • 你需要进入真实工程，而不是继续停留在单点知识

阅读目标

读完这一部分，你应该能做到：

  • 通过一个小而完整的案例串起数据结构、错误处理和资源管理
  • 在陌生项目中快速识别常见工程模式
  • 建立从入口点、构建脚本、第三方边界到核心调用链的阅读路径

怎么读这一部分

建议按下面顺序阅读：

  1. 先读 第12章，把前面概念收束成一个完整小项目。
  2. 再读 第13章，建立模式识别能力。
  3. 最后读 第14章，把方法扩展到大型仓库。

读到卡住时跳哪里

  • 小项目里卡在 variant / string_view / 错误链：回看 第7章、第4章、第6章
  • 大项目里卡在模板、宏、UB：回看 第三部分
  • 想现场配置工具：跳到 附录 E
